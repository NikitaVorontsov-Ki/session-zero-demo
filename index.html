MCP — Model Context Protocol hide
An MCP server is a process (local or remote) that exposes tools to agents.
  • Declares a tool manifest — names, input schemas, descriptions
  • Implements each tool — API calls, auth, retries, pagination
  • Speaks JSON-RPC over stdio or HTTP/SSE
  • Sandboxes secrets — tokens never enter agent context

Discovery: at startup, the runtime queries each server for its manifest and injects tool schemas into the agent's system prompt.

Flow: agent emits tool_use → runtime routes by name → server hits real API → JSON result → agent sees tool_result.

Errors: come back as tool_result with is_error: true. The agent decides whether to retry, adjust args, or surface the failure.

Note: MCP also defines resources and prompts as primitives. This demo uses only tools.
